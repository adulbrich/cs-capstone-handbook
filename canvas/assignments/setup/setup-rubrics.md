# Setup - Rubrics

## Executive Summary (25 pts)

- **Exceeds Expectations** (25 to >20.0 pts): 4–6 bullets (≤120 words) that explicitly note: repo created, comms chosen, board live, 5–10 tasks queued, partner sync scheduled, and, if inherited, repo summary (langs/frameworks, build/run, services, tests Y/N, last tag, open issues, default branch), run verified (exact command(s) + outcome; small screenshot/log OK), gaps noted with ≥2 follow-up tasks on board, access/permissions confirmed.
- **Meets Expectations** (20 to >5.0 pts): Summary present but 1–2 items vague/missing, and, if inherited, repo summary mostly complete; run command(s) + outcome noted; ≥1 follow-up task or access/permissions stated.
- **Does Not Meet Expectations** (5 to >0 pts): Missing or unclear, and, if inherited, no summary or no run evidence, no follow-ups, access not addressed.

## Communication Mediums and Cadences (20 pts)

- **Exceeds Expectations** (20 to >16.0 pts): Submitted table lists primary async channel + escalation email, standup cadence, partner sync window (day/time), and response targets (e.g., PR review ≤24h).
- **Meets Expectations** (16 to >4.0 pts): Channels + at least two of {cadence, partner window, response targets}.
- **Does Not Meet Expectations** (4 to >0 pts): Channels unclear; cadence/partner window missing.

## Repo and Basics (20 pts)

- **Exceeds Expectations** (20 to >16.0 pts): Repo exists; README includes one-liner project description + team roster/contacts; .gitignore present; LICENSE present or noted “pending partner confirmation”; teammates + partner invited (roles stated). Report includes repo & README screenshot.
- **Meets Expectations** (16 to >4.0 pts): Repo + README (one-liner + roster) and .gitignore; most invites sent; screenshot present.
- **Does Not Meet Expectations** (4 to >0 pts): Repo/README missing; invites not set.

## PM Rails (Board and Next Tasks/Plan) (25 pts)

- **Exceeds Expectations** (25 to >20.0 pts): Project board has columns ≈ Backlog → Ready → In Progress → Review → Done. ≥5 items visible with owners and due dates. Screenshot shows items not all in Backlog.
- **Meets Expectations** (20 to >5.0 pts): Board exists with columns; ≥5 items present OR owners/dates on most.
- **Does Not Meet Expectations** (5 to >0 pts): No board, <5 items, no owners/dates.

## Branching and Reviews (10 pts)

- **Exceeds Expectations** (10 to >8.0 pts): README (or report) documents feature/* → PR → ≥1 review → merge; PR template path shown (.github/PULL_REQUEST_TEMPLATE.md) if available; inherited teams note any existing contribution guidelines they will follow.
- **Meets Expectations** (8 to >2.0 pts): Flow documented; PR checklist or template mentioned (even minimal).
- **Does Not Meet Expectations** (2 to >0 pts): No documented flow; no template/checklist.
